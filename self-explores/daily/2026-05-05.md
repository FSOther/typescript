## Key learnings

### TypeScript Core (M02)
- Union types, type inference, tuples, enums, literal types, type aliases
- null/undefined handling: optional chaining `?.`, nullish coalescing `??`
- `unknown` vs `any`: unknown cần type guard trước khi dùng

### Classes & Interfaces (M06)
- Abstract classes, `implements` pattern
- `interface` vs `type alias`: interface extendable, type alias more flexible
- Access modifiers: `private`, `protected`, `readonly`

### Advanced Types (M07+M08)
- Intersection `&`, discriminated unions, type guards (`in`, `instanceof`, predicates)
- Function overloads, `as const`, `satisfies` operator
- Generic constraints: `<T extends SomeType>`

### Type Utilities (M10)
- `typeof` (value → type), `keyof` (type → union of keys)
- Mapped types: `{ [K in keyof T]: V }` + modifiers `?`, `-?`, `readonly`
- Conditional types + `infer`, built-in utilities (Partial, Pick, Omit, ReturnType...)

### DnD Project (M13) — BUILD mode
- Abstract generic base class: `abstract class Component<T extends HTMLElement, U extends HTMLElement>`
- Singleton: `private constructor()` + `static getInstance()`
- Observer pattern: `type Listener<T> = (items: T[]) => void`
- `@autobind` decorator avoids `.bind(this)` in constructors
- `event.preventDefault()` in `dragOverHandler` = REQUIRED for HTML5 DnD

### React + TypeScript (M17)
- Props: `function Comp({ prop }: Props)` — NOT `FC<Props>` (older pattern)
- `useState<Goal[]>([])` — explicit generic needed for empty array
- `useRef<HTMLInputElement>(null)` — explicit generic, not inferable from null
- Form events: `FormEvent<HTMLFormElement>`

### Node.js + Express (M18)
- `@types/node` + `@types/express` = required separate installs
- Import with `.js` extension even for `.ts` source (Node resolves compiled output)
- `verbatimModuleSyntax: true` → `import { type X }` required for type-only imports
- Native strip-only: drops annotations only (no enum/decorator support)
- Bun: full TS support, Node-compatible
