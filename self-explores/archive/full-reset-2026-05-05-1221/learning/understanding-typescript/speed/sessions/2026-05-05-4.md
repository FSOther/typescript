### L119-L121 — `typeof` Operator (DEEP_DIVE)

**80/20:** TypeScript's `typeof` derives a TYPE from a VALUE. Eliminates manual type re-declaration for existing values.

**Must-know:**
```typescript
const settings = {
  difficulty: "easy",
  minLevel: 10,
  didStart: false,
  players: ["Alice", "Bob"]
}

type Settings = typeof settings
// Settings = { difficulty: string; minLevel: number; didStart: boolean; players: string[] }

function loadData(settings: Settings) { ... }
// No need to manually write the type
```

**Also useful for function types:**
```typescript
function sum(a: number, b: number) { return a + b }
function subtract(a: number, b: number) { return a - b }

type SumFn = typeof sum  // (a: number, b: number) => number

function performMathAction(cb: SumFn | typeof subtract) { ... }
```

- `typeof` in TS = derive type from value (not same as JS `typeof` which returns string at runtime)
- Works on variables, constants, functions
- On `const` → narrows to literal type; on `let` → widens to base type

**Active Recall:**
1. Q: `typeof` operator dùng để làm gì? → A: Derive TypeScript type từ existing value — tránh phải viết lại type thủ công
2. Q: `const x = "hello"` → `type T = typeof x` — T là gì? → A: `"hello"` (literal type, vì const không thể reassign)
3. Q: `let x = "hello"` → `type T = typeof x` — T là gì? → A: `string` (widened, vì let có thể reassign)

### L122-L123 — `keyof` Operator (DEEP_DIVE)

**80/20:** `keyof SomeType` = union of all property names (as string literals). Combine with generics to constrain keys.

**Must-know:**
```typescript
type User = { name: string; age: number }
type UserKeys = keyof User  // "name" | "age"

// The CRITICAL pattern: constrain key param to valid keys of object
function getProp<T extends object, K extends keyof T>(obj: T, key: K): T[K] {
  return obj[key]
}

const user = { name: "Max", age: 36 }
const name = getProp(user, "name")  // T[K] inferred as string
const age = getProp(user, "age")    // T[K] inferred as number
getProp(user, "foo")  // ERROR: "foo" is not keyof typeof user
```

- `keyof` = TypeScript-only (no JS equivalent)
- Returns union type of property name strings
- `K extends keyof T` = constraint linking two generic params

**Active Recall:**
1. Q: `keyof User` khi `User = { name: string; age: number }` → A: `"name" | "age"` (union of literal strings)
2. Q: `K extends keyof T` trong generic function nghĩa là gì? → A: K phải là một trong các property names của T → TypeScript sẽ error nếu truyền key không tồn tại
3. Q: Return type `T[K]` là gì? → A: Indexed access type — type của value tại key K trong object T (VD: nếu K="age", T[K] = number)

### L124-L125 — Indexed Access Types (DEEP_DIVE)

**80/20:** `SomeType["propName"]` = extract the type of a specific property. `SomeArray[number]` = extract element type of array.

**Must-know:**
```typescript
type AppUser = {
  name: string
  age: number
  permissions: { id: string; title: string; description: string }[]
}

// Extract sub-type without copy-paste
type Perms = AppUser["permissions"]  // { id: string; title: string; description: string }[]

// Extract element type from array type
type Perm = Perms[number]  // { id: string; title: string; description: string }

// Simple case
type Names = string[]
type Name = Names[number]  // string
```

**Two forms:**
1. `ObjType["propName"]` = type of that property
2. `ArrayType[number]` = type of elements in array

**Active Recall:**
1. Q: `AppUser["permissions"]` syntax làm gì? → A: Extracts type của property "permissions" từ AppUser type
2. Q: `string[][number]` là gì? → A: `string` — element type of string array
3. Q: Tại sao dùng indexed access thay copy-paste? → A: Tránh maintenance burden — nếu original type thay đổi, derived type tự update

### L126-L127 — Mapped Types (DEEP_DIVE)

**80/20:** Mapped type = `{ [K in keyof T]: NewType }` — create new object type by transforming all keys of another type.

**Must-know:**
```typescript
type Operations = {
  add: (a: number, b: number) => number
  subtract: (a: number, b: number) => number
}

// Map operations → results (same keys, different value type)
type Results<T> = {
  [K in keyof T]: number
}

const mathResults: Results<Operations> = {
  add: 5,      // OK
  subtract: 3  // OK
}
```

**Modifiers:**
```typescript
// Make all optional
type Optional<T> = { [K in keyof T]?: T[K] }

// Remove optional (-?)
type Required<T> = { [K in keyof T]-?: T[K] }

// Make all readonly
type Readonly<T> = { readonly [K in keyof T]: T[K] }

// Remove readonly (-readonly)
type Mutable<T> = { -readonly [K in keyof T]: T[K] }
```

**Active Recall:**
1. Q: Mapped type syntax? → A: `{ [K in keyof T]: ValueType }` — iterate over keys, assign new value type
2. Q: `[K in keyof T]?:` vs `[K in keyof T]-?:` — difference? → A: `?:` makes optional; `-?:` REMOVES optional (makes required)
3. Q: `readonly [K in keyof T]` vs `-readonly [K in keyof T]` → A: `readonly` adds readonly; `-readonly` removes it

### L128 — Template Literal Types (STUDY)

**80/20:** `` `${UnionA}-${UnionB}` `` in type position = create all combinations as string literal union. Combines with `keyof` for dynamic property names.

**Must-know:**
```typescript
type ReadPerms = "no-read" | "read"
type WritePerms = "no-write" | "write"

// All combinations
type FilePerms = `${ReadPerms}-${WritePerms}`
// = "no-read-no-write" | "no-read-write" | "read-no-write" | "read-write"

// With keyof: create event handler names
type DataFile = { data: string; permissions: FilePerms }
type EventNames = `${keyof DataFile}Changed`  // "dataChanged" | "permissionsChanged"
```

**Active Recall:**
1. Q: `` `${A}-${B}` `` khi A và B là union types → result là gì? → A: Union của ALL combinations (cartesian product)
2. Q: `` `${keyof T}Changed` `` tạo ra gì? → A: Union of strings: mỗi property name của T + "Changed"
3. Q: Template literal type là TS hay JS? → A: TypeScript — strips at compile time, không tồn tại ở runtime

### L129-L130 — Conditional Types (STUDY)

**80/20:** `T extends SomeType ? TypeIfTrue : TypeIfFalse` — ternary expression for types. Used in utility types.

**Must-know:**
```typescript
// Utility type: extract element type, works on arrays OR non-arrays
type GetElementType<T> = T extends (infer _)[] ? T[number] : never
// If T is array → return element type; else → never

// Another use: conditional return type
type FullnamePerson = { firstName: string; lastName: string }
type FullNameOrNever<T> = T extends FullnamePerson ? string : never

function getFullName<T extends object>(person: T): FullNameOrNever<T> {
  if ("firstName" in person && "lastName" in person) {
    return `${(person as any).firstName} ${(person as any).lastName}` as FullNameOrNever<T>
  }
  throw new Error("No firstname and/or last name found")
}

getFullName({})                          // never
getFullName({ firstName: "Max", lastName: "S" })  // string
```

**Active Recall:**
1. Q: Conditional type syntax? → A: `T extends SomeType ? TrueType : FalseType` — ternary but in type land
2. Q: Khi nào dùng conditional types? → A: Building utility/helper types, library code — ít khi cần trong app code
3. Q: `never` trong else branch của conditional type có ý nghĩa gì? → A: Biểu thị "không thể sử dụng type này trong nhánh đó" — safe fallback

### L131 — `infer` Keyword (STUDY)

**80/20:** `infer R` trong conditional type = extract a sub-type from the checked type. Use `R` on the true branch.

**Must-know:**
```typescript
// Extract return type of any function
type ReturnValueType<T> = T extends (...args: any[]) => infer R ? R : never

function add(a: number, b: number) { return a + b }
type AddFn = typeof add  // (a: number, b: number) => number

type AddReturnType = ReturnValueType<AddFn>  // number
```

- `infer R` = "capture whatever type is here as R"
- Must be inside a conditional type (`extends ... ? ...`)
- Use `R` only on the true branch (after `?`)

**Active Recall:**
1. Q: `infer` keyword làm gì? → A: Extract (capture) a sub-type from the type being checked — like a type variable inside condition
2. Q: `T extends (...args: any[]) => infer R` — `R` là gì? → A: Return type của function — TypeScript "infers" it from the actual function type
3. Q: Có thể dùng `infer R` ở else branch (`after :`) không? → A: Không — `infer` captured variables chỉ accessible ở true branch

### L132 — Built-in Utility Types (STUDY)

**80/20:** TypeScript has built-in utility types that use the features above. Know them — don't build from scratch.

**Key built-ins:**
```typescript
// Property modifiers
Partial<T>        // all props optional
Required<T>       // all props required (removes ?)
Readonly<T>       // all props readonly
Mutable<T>        // all props non-readonly (not built-in, but common pattern)

// Object manipulation
Pick<T, K>        // keep only keys K from T
Omit<T, K>        // remove keys K from T
Record<K, V>      // { [k in K]: V }

// Union/type manipulation
Exclude<T, U>     // remove U from union T
Extract<T, U>     // keep only U in union T
NonNullable<T>    // remove null | undefined from T

// Function types
ReturnType<T>     // return type of function T
Parameters<T>     // tuple of parameter types of function T

// Others
Awaited<T>        // unwraps Promise<T> → T
```

**Active Recall:**
1. Q: `Partial<{ name: string; age: number }>` → A: `{ name?: string; age?: number }` — all optional
2. Q: `Pick<User, "name" | "age">` vs `Omit<User, "id">` — difference? → A: Pick = KEEP listed keys; Omit = REMOVE listed keys
3. Q: `ReturnType<typeof someFunction>` — dùng khi nào? → A: Khi cần extract return type của existing function mà không muốn viết lại manually

```typescript
typeof value          // derive type FROM value
```

```typescript
// Mapped type
```

**Use built-ins first:**
- `Partial<T>`, `Required<T>`, `Readonly<T>`
- `Pick<T, K>`, `Omit<T, K>`, `Record<K, V>`
- `ReturnType<T>`, `Parameters<T>`, `Awaited<T>`

**Key insight:** These features are primarily for **library/framework authors** and **advanced utility types** — not required for typical app code. Learn to RECOGNIZE them; only build custom ones when built-ins don't cover your case.
